# ARAAK AI Command Center

## الغرض

تحويل إدارة المحتوى التنموي وتشغيل أكاديمية أراك من أعمال متفرقة تعتمد على المتابعة اليدوية إلى منظومة تشغيل ذكية داخل **CEO Office 360**، مع الاحتفاظ بالاعتماد البشري للقرارات الحساسة.

الاسم التشغيلي المقترح:

- **ARAAK NAMA AI** — منظومة نماء الذكية.
- **ARAAK AI Command Center** — مركز القيادة داخل مكتب الرئيس التنفيذي.

## المبدأ الحاكم

المنظومة لا تمنح نموذج الذكاء الاصطناعي وصولاً مباشراً غير مقيد إلى حسابات النشر أو بيانات المتدربين. جميع العمليات تمر عبر طبقة تنفيذ وصلاحيات وسجل تدقيق.

```text
الهدف أو الفرصة
      ↓
المشرف التنفيذي الذكي
      ↓
وكلاء متخصصون
      ↓
تدقيق وهوية وحوكمة
      ↓
اعتماد أخضر / أصفر / أحمر
      ↓
طبقة التنفيذ والموصلات
      ↓
النشر أو تشغيل البرنامج
      ↓
التحليل والتحسين
```

## الوكلاء في الإصدار الأول

| الوكيل | المسؤوليات | صلاحية التنفيذ |
|---|---|---|
| المشرف التنفيذي الذكي | توزيع المهام، مراجعة النتائج، التصعيد، الموجز التنفيذي | لا ينشر مباشرة |
| الاستراتيجية والتحرير | التقويم التحريري، الحملات، الجمهور والرسالة | إنشاء مسودات وخطط |
| الرصد والفرص | الأخبار، المنح، الشراكات، فرص البرامج والمشاريع | قراءة وتحليل |
| صناعة المحتوى | المقالات والمنشورات والنشرات ونصوص الفيديو | إنشاء مسودات |
| التدقيق والحوكمة | اللغة والحقائق والهوية والحقوق والحساسية | اعتماد أولي وتصنيف |
| النشر والتوزيع | الجدولة والنشر والتحديث والإرسال | بعد صدور الاعتماد |
| التحليل والتحسين | الأداء والتحويل والتوصيات | قراءة بيانات المنصات |
| أكاديمية أراك | البرامج والمدربون والمتدربون والجودة والأثر | ضمن الصلاحيات الأكاديمية |

## مستويات الاعتماد

### أخضر — اعتماد آلي

- محتوى معرفي من مادة معتمدة.
- تذكير ببرنامج قائم دون تغيير السعر أو الشروط.
- إعادة استخدام محتوى سابق بعد فحص التكرار.
- تقارير تشغيل داخلية لا تتضمن بيانات حساسة.

### أصفر — اعتماد إداري

- إطلاق حملة برنامج.
- إعلان سعر أو خصم.
- ذكر شريك أو مدرب.
- نشرات الشركاء والعملاء.
- تعديلات على صفحات البرامج.

### أحمر — اعتماد تنفيذي أو مختص

- موقف أو تصريح باسم المجموعة.
- محتوى قانوني أو ديني أو سياسي حساس.
- اعتماد أو شهادة أو ادعاء رسمي.
- رد على أزمة أو شكوى.
- تغيير مالي أو تعاقدي أو بيانات حساسة.

## الوحدات الوظيفية

### 1. لوحة القيادة

- حالة الوكلاء.
- قائمة الانتظار.
- المواد المنتجة والمنشورة.
- البرامج النشطة.
- القرارات المطلوبة.
- سلامة الموصلات والخدمات.

### 2. غرفة عمليات المحتوى

- بنك الأفكار.
- الخطة التحريرية.
- إنتاج متعدد القنوات.
- مكتبة التصاميم والأصول.
- المراجعة والاعتماد.
- الجدولة والنشر.
- قياس التحويل.

### 3. تشغيل أكاديمية أراك

- تصميم البرامج ومخرجات التعلم.
- صفحات البرامج والحملات.
- التسجيل والمدفوعات.
- خدمة المتدربين.
- تنسيق المدربين.
- الحضور والتقييم.
- الجودة وIQA.
- الشهادات والتقارير.
- قياس الأثر بعد 30 و90 و180 يوماً.

### 4. العقل المؤسسي

مصادر المعرفة المقترحة:

- استراتيجية مجموعة أراك.
- الرؤية والرسالة والقيم.
- الهوية البصرية وقواعد الصياغة.
- تعريف الوحدات والقطاعات.
- البرامج والحقائب التدريبية.
- التقارير والمشاريع السابقة.
- سياسات الجودة والاعتماد وIQA.
- قوائم المدربين والشركاء والخبراء.
- أرشيف المحتوى ونتائج الأداء.

## التقنية المقترحة

### الواجهة

- React + TypeScript + Vite.
- Tailwind CSS.
- مكونات متوافقة مع السمات الحالية لمكتب الرئيس التنفيذي.

### الخلفية

- Express في الإصدار الانتقالي الحالي.
- يوصى بفصل خدمات الوكلاء لاحقاً داخل FastAPI أو خدمة Node مستقلة.
- PostgreSQL للبيانات التشغيلية.
- pgvector أو خدمة بحث متجهي موثوقة لقاعدة المعرفة.
- Redis وطابور مهام للعمليات الطويلة.

### الذكاء الاصطناعي

- OpenAI Responses API.
- Agents SDK لإدارة التوجيه بين الوكلاء والتتبع.
- Structured Outputs لتوحيد مخرجات المحتوى والاعتماد.
- File Search أو قاعدة معرفة خاصة بالمؤسسة.
- حواجز إدخال وإخراج قبل أي إجراء تنفيذي.

### التكاملات

- الموقع الرسمي وأكاديمية أراك.
- البريد والنشرات.
- WhatsApp Business.
- LinkedIn وX وInstagram حسب إمكانات واجهاتها الرسمية.
- LMS.
- Zoom أو Google Meet.
- بوابة الدفع.
- إصدار الشهادات.
- التخزين السحابي.

### المرحلة 1 — النواة المرئية والحوكمة

- شاشة مركز القيادة.
- تعريف الوكلاء.
- خط إنتاج المحتوى.
- برامج الأكاديمية.
- مركز الاعتمادات.
- سجل الأعمال.

### المرحلة 2 — API وقاعدة البيانات

- جداول المحتوى والبرامج والاعتمادات.
- نقاط نهاية فعلية.
- تسجيل عمليات الوكلاء.
- صلاحيات حسب الدور.

### المرحلة 3 — قاعدة المعرفة والذكاء الحقيقي

- رفع ملفات أراك.
- البحث الدلالي.
- تشغيل OpenAI تحت الإشراف.
- التقييم الآلي لجودة المخرجات.

### المرحلة 4 — الموصلات

- الموقع والبريد.
- قنوات التواصل.
- LMS والتسجيل.
- WhatsApp.
- النشر المجدول.

### المرحلة 5 — الاستقلال التشغيلي المضبوط

- فتح النشر الأخضر فقط.
- قياس الأخطاء والدقة.
- زيادة الصلاحيات تدريجياً.
- إبقاء الأحمر تحت الاعتماد التنفيذي دائماً.

## معيار نجاح النسخة الأولى

- تقليل زمن إنتاج المحتوى المعتمد بنسبة لا تقل عن 50%.
- تنفيذ 90% من الخطة التحريرية الشهرية.
- خفض المواد المرتجعة بسبب اللغة أو الهوية.
- ربط كل مادة بهدف ومؤشر تحويل.
- وجود ملف جودة كامل لكل برنامج أكاديمي.
- عدم تنفيذ أي إجراء حساس دون اعتماد مسجل.
